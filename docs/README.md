# ドキュメントアセット概要

> ファイル: docs/README.md ｜ 役割: docs 配下の索引 ｜ 目的: 各アセットの内容と更新手順をワンストップで把握する

## 1. talklog_withgpt.md

- **内容**: キャリア戦略について GPT と行った対話全文。質問・回答・参考リンクが時系列で記録されている。
- **用途**: 新しいインサイトや TODO を抽出し、`strategy/` や `job-hunt/` の具体的なアクションへ転記する際の一次情報源。
- **更新手順**:
  1. 重要度の高い対話のみを追記し、日付とトピックを簡潔に見出し化する。
  2. 抽出したアクションは関連ディレクトリへ移し、ログ側には「転記済み」メモを添える。
  3. 年月ごとに水平線を挿入し、読み返しやすいチャンクに分割する。

## 2. works.csv（実績データベース）

- **内容**: プロジェクトごとの概要、成果指標、スキル、参照リンクを含むマスター CSV。列は `名前 / Actions / Outcome / Skills / Tools ...` といった定義で固定。
- **用途**: レジュメ作成、ポートフォリオのソース、スキルエビデンス作成時の参照軸。
- **更新手順**:
  1. 新規プロジェクトは必須列（名前・Year・Type・Roles・Outcome）を先に埋めて保存。
  2. KPI や Before/After が未確定の場合は `TBD` を入れ、確定後に必ず置き換える。
  3. `Artifacts` 列に Figma / GitHub / 自サイトなどの参照リンクを箇条書きで追記。

## 3. works-article/

- **内容**: 各実績のロングフォーム原稿と添付メディア。フォルダ名は `プロジェクト名 + ハッシュ` で揃え、配下に `.md` と素材を格納している。
- **用途**: ポートフォリオサイトや選考資料のコピーベース。CSV の `Artifacts` と相互参照する。
- **更新手順**:
  1. `works.csv` に登録した「名前」と同じタイトルで Markdown を作成。
  2. セクション構成（Basic Information → Overview → Problem → Actions → Outcome → Retrospective）を踏襲し、更新日を明記。
  3. メディア資産を同フォルダに保存し、本文から相対パスで参照する。

## 4. skill_evidence_v3.csv（スキルエビデンス）

- **内容**: デザイン人材スキルマトリクスのカテゴリごとに、推定レベルと根拠プロジェクト、Did/Result を記録した一覧。
- **用途**: 面接質問への即答準備、スキル棚卸し、評価シート作成。
- **更新手順**:
  1. `works.csv` で実績を追加したら、該当スキル行に `根拠プロジェクト` を追記。
  2. プロジェクトごとの成熟度に合わせて `推定レベル` を 1-5 で更新し、理由を `Did/Result` に簡潔に書く。
  3. レベル変更時は日付をコメント（例: `// 2025-02-14 update`）として残すと経緯が追いやすい。

## 5. skillmap_v2_actions_retrospective.csv（レトロスペクティブビュー）

- **内容**: スキル単位で「推定レベル」「根拠プロジェクト数」「根拠プロジェクト名」をまとめたスナップショット。
- **用途**: スキルの偏りやエビデンス数の不足を素早く把握し、次のアクション（深掘り or 新規実績）の意思決定に使う。
- **更新手順**:
  1. `skill_evidence_v3.csv` を更新したタイミングでこのファイルを見直す。
  2. 数値は手作業更新のため、差分管理のしやすい順序（カテゴリ→技術名）を崩さない。
  3. 空のスキルには `0` と `-` を明示し、未経験領域との区別を付ける。

## 6. designer-skill-matrix.csv（リファレンス）

- **内容**: Concent Inc. が公開しているデザイン人材スキルマトリクス最新版のローカルコピー。
- **用途**: スキルレベル判定時の共通言語。`skill_evidence_v3.csv` や面接資料で引用する。
- **更新手順**:
  1. 公開記事がアップデートされたら最新 CSV をダウンロードし、既存ファイルを差し替える。
  2. 列・行の順序が変わった場合は、他ファイルの参照先も合わせて更新する。
  3. 版数と取得日を冒頭コメントに追記し、どの年度のマトリクスか分かるようにする。
