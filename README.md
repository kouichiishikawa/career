# キャリア管理ディレクトリ

> ファイル: README.md ｜ 役割: ルートガイド ｜ 目的: ディレクトリ全体の概要と更新ルールを即座に確認する

転職・キャリア形成に関する情報を一元管理するためのディレクトリです。最初にこのファイルを読み、目的のディレクトリへ移動してください。

## ディレクトリ構造

### 📋 profile/ - プロフィール情報
自分自身の情報を整理・管理します。

- **resume/**: 履歴書・職務経歴書のバージョン管理
  - 命名例: `resume_2025-01.pdf`, `resume_engineer.md`
- **introduction.md**: 採用プラットフォーム用の自己紹介文（汎用版）
- **career-goal.md**: キャリアゴール、志望動機、面接対策の核となるドキュメント
- **career-history.md**: これまでのキャリアの振り返り、学んだこと
- **skills-inventory.md**: 現在のスキルセット、得意分野、実績の棚卸し

### 🎯 job-hunt/ - 転職活動
実際の転職活動を管理します。

- **active/**: 現在応募中・選考中の企業
  - 企業ごとにフォルダを作成: `active/[企業名]/`
  - 含める情報: 応募書類、面接メモ、企業研究、やり取り記録
- **archive/**: 過去の転職活動記録
  - 時期ごとに整理: `archive/2025-01_job-search/`
- **templates/**: 再利用可能な応募書類テンプレート
  - カバーレター、自己PR、志望動機のベース

### 📚 learning/ - スキルアップ
継続的な学習とスキル向上を記録します。

- **study-plan.md**: 学習計画、習得したい技術・スキル
- **certifications/**: 取得した資格、認定証の保管

### 🤝 networking/ - 人脈・ネットワーク
人脈やコネクションを管理します。

- 人脈リスト、紹介者情報
- 勉強会・イベント参加記録
- LinkedIn等のコネクション管理

### 📊 strategy/ - 戦略・計画
中長期的な戦略と計画を立てます。

- **market-research/**: 業界動向、市場調査、求人トレンド
- **career-plan/**: キャリアビジョン、5年後・10年後の目標

### 🗂 docs/ - キャリアアセット集
GPT との対話ログや実績・スキルデータを集約します。詳細な運用フローは `docs/README.md` にまとめています。

- **talklog_withgpt.md**: キャリア設計に関する GPT との全履歴。新しい示唆を得たら `strategy/` 配下へ転記し、ログ側には要約を追加する。
- **works.csv**: プロジェクト実績の母集計。列構成（名前 / Actions / KPI Metrics など）を固定し、`docs/works-article/` の原稿や `skill_evidence_v3.csv` と紐付ける。
- **works-article/**: ポートフォリオ用原稿と添付メディア。フォルダ名は `works.csv` の「名前」とハッシュで揃え、`Artifacts` 列にパスを記載。
- **skill_evidence_v3.csv**: デザイン人材マトリクス準拠のスキル評価。各行に「推定レベル」「Did」「Result」を記載し、レビューで即答できるよう根拠を更新。
- **skillmap_v2_actions_retrospective.csv**: スキルごとのレベルと根拠件数を集約した表。強み／弱みの偏りを把握し、面接対策の優先度付けに使う。
- **designer-skill-matrix.csv**: Concent 公開の最新スキル指標。改訂版が出たらここを更新し、他 CSV のレベル基準を再評価する。

## 使い方

### 転職活動を始めるとき
1. `profile/career-goal.md`でキャリアゴールを明確化
2. `profile/introduction.md`で自己紹介文を作成
3. `docs/`配下のスキル・実績データを最新化
4. `job-hunt/active/`で応募企業を管理

## 企業エントリーから面談対策までのワークフロー

### 📊 フロー全体像

```
1. スカウト/求人発見
   ↓
2. 企業研究（company-research.md作成）
   ↓
3. 志望動機作成（motivation-letter.md作成）
   ↓
4. 応募書類準備・提出
   ↓
5. 面談準備（interview-prep.md作成）
   ↓
6. 面談実施
   ↓
7. 振り返り（interview-notes.md作成）
   ↓
8. 次ステップへ or アーカイブ
```

---

### STEP 1: 企業ディレクトリの作成

企業ごとに専用ディレクトリを作成します。

```bash
job-hunt/active/[企業名]/
├── company-research.md      # 企業研究
├── motivation-letter.md     # 志望動機
├── interview-prep.md        # 面談準備
├── interview-notes.md       # 面談記録（複数回の場合は連番）
└── communications/          # メール等のやり取り
```

**テンプレートのコピー方法:**
```bash
cp -r job-hunt/templates/* job-hunt/active/[企業名]/
```

---

### STEP 2: 企業研究（company-research.md）

#### 📝 記入項目
- 基本情報（企業名、URL、事業内容、従業員数等）
- プロダクト/サービス分析
- ビジョン・ミッション
- デザイン組織の状況
- カルチャー・働き方
- 魅力ポイント・懸念点

#### 🔍 情報収集先
- コーポレートサイト、採用ページ
- Wantedly、Note、X (Twitter)
- 口コミサイト（OpenWork、転職会議等）
- プロダクトを実際に使ってみる

#### 🤖 AIエージェント活用
Claude Codeで以下を依頼:
```
以下の企業について調査してください：
- 企業名: [企業名]
- 募集ページ: [URL]

company-research.mdのテンプレートに沿って情報を整理してください。
```

---

### STEP 3: 志望動機作成（motivation-letter.md）

#### 📝 作成プロセス

**1. 企業の魅力を整理（Why this company?）**
- `company-research.md`から魅力ポイントを抽出
- 企業のビジョン・ミッションへの共感を言語化

**2. 自分が貢献できることを整理（What I can do?）**
- `/profile/career-goal.md`の「実現するための強み」を参照
- `/docs/works.csv`から関連する実績を選定
- `/docs/skill_evidence_v3.csv`からスキルエビデンスを抽出

**3. キャリアゴールとの整合性を確認（Why now?）**
- `/profile/career-goal.md`の「なぜその方向を目指すのか」を参照
- この企業だからこそ実現できることを明確化

**4. 想定質問への回答を準備**
- テンプレートの想定質問リストを埋める
- `/profile/career-goal.md`の「想定Q&A」を参考に

#### 🤖 AIエージェント活用
Claude Codeで以下を依頼:
```
以下のドキュメントを参照して、[企業名]への志望動機を作成してください：
- /profile/career-goal.md
- /docs/works.csv
- /docs/skill_evidence_v3.csv
- job-hunt/active/[企業名]/company-research.md

motivation-letter.mdのテンプレートに沿って作成してください。
```

---

### STEP 4: 応募書類準備

#### 📄 必要書類
- 履歴書（`profile/resume/`から最新版を使用）
- 職務経歴書（必要に応じてカスタマイズ）
- ポートフォリオ（URL or PDF）
- カバーレター（`motivation-letter.md`から抽出）

---

### STEP 5: 面談準備（interview-prep.md）

#### 📝 準備項目
- 面談の目的・流れの確認
- 自己紹介スクリプト作成
- 経歴・実績の説明準備（優先順位順に3つ）
- 志望動機の整理
- 想定質問への回答準備
- 逆質問リスト作成

#### 🔗 参照ドキュメント
- `/profile/career-goal.md` - キャリアストーリー、想定Q&A
- `./motivation-letter.md` - 志望動機
- `/docs/works.csv`, `/docs/works-article/` - 実績詳細

#### 🤖 AIエージェント活用
Claude Codeで以下を依頼:
```
[企業名]の[面談種別]の準備をしたいです。
以下のドキュメントを参照して、interview-prep.mdを作成してください：
- /profile/career-goal.md
- job-hunt/active/[企業名]/motivation-letter.md
- /docs/works.csv
```

---

### STEP 6: 面談実施

#### ✅ 当日の流れ
1. 自己紹介（5分）
2. 経歴・実績の説明（10分）
3. 志望動機（5分）
4. 質疑応答（15分）
5. 逆質問（10分）

#### 📝 メモ
- 面談中はメモを取る（質問内容、回答、企業の説明等）
- カルチャーフィットを意識して観察
- 次のステップを確認

---

### STEP 7: 振り返り（interview-notes.md）

#### 📝 記録項目
- 面談内容（企業説明、質疑応答、自分からの質問）
- 所感・評価（良かった点、懸念点、カルチャーフィット）
- 次のアクション（お礼メール、追加確認事項）
- 学び・改善点（うまく答えられなかった質問、次回への改善点）

#### ⏰ 実施タイミング
- 面談終了後、**24時間以内**に記録
- お礼メールも24時間以内に送信

#### 🤖 AIエージェント活用
Claude Codeで以下を依頼:
```
面談メモを整理して、interview-notes.mdのテンプレートに沿ってまとめてください。

【面談メモ】
[箇条書きでメモを貼り付け]
```

---

### STEP 8: 次ステップへ or アーカイブ

#### ✅ 選考通過の場合
- 次回面談の準備を開始
- `interview-prep.md`（連番）を新規作成

#### ❌ 不採用の場合
- 学びを`interview-notes.md`に追記
- ディレクトリを`job-hunt/archive/YYYY-MM_[企業名]/`へ移動
- 次回に活かすポイントを`profile/career-goal.md`に反映

## ドキュメント間の関連性

### 🎯 コアドキュメント
- **profile/career-goal.md** - すべての志望動機・面接対策の核
- **docs/works.csv** - 実績データの母集計
- **docs/skill_evidence_v3.csv** - スキルエビデンス

### 📄 派生ドキュメント
- **job-hunt/active/[企業名]/motivation-letter.md**
  - `profile/career-goal.md`から企業別にカスタマイズ
- **job-hunt/active/[企業名]/interview-prep.md**
  - `motivation-letter.md`と`career-goal.md`を統合

### 🔄 更新の流れ
```
新規実績 → docs/works.csv更新
         → docs/works-article/詳細記事作成
         → docs/skill_evidence_v3.csv更新
         → profile/career-goal.md反映（必要に応じて）
```

| 種類 | 頻度 | 所要時間 | 目的 |

### 🔄 随時メンテナンス：新規実績の追加

**トリガー**: 新しいプロジェクトが完了した時

#### ステップ

1. **`docs/works.csv`に実績を追加**
   - 基本情報（名前、期間、タイプ、ロール等）
   - KPI（Before → After）
   - 課題、アクション、成果

2. **`docs/works-article/`に詳細記事を作成**
   - ポートフォリオ用の詳細な記事
   - プロジェクト背景、プロセス、成果、学びを記載
   - 画像・図表も含める

3. **`docs/skill_evidence_v3.csv`を更新**
   - 該当スキルの「Did」「Result」を追加
   - レベルの見直し（必要に応じて）

4. **`profile/career-goal.md`に反映（必要に応じて）**
   - 「実現するための強み」セクションに追加
   - 想定Q&Aに新しい実績を組み込む

## メンテナンスポリシー

### 基本原則

- 新規プロジェクトは `docs/works.csv` → `docs/works-article/` → `docs/skill_evidence_v3.csv` の順で更新すると整合性が取りやすい
- `talklog_withgpt.md` は重要な示唆のみ追記し、アクションやタスクリストは `strategy/` 配下へ移して重複を避ける
